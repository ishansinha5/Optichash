How it works (Phase 2): I am currently rebuilding the ML pipeline from the ground up. The original iteration was a collaborative filtering model trained on a massive dataset I scraped from Goodreads and Amazon[cite: 19]. The new architecture is built around compute optimization[cite: 19]. I'm deploying a PyTorch vision model wrapped in a Python FastAPI endpoint[cite: 19]. To keep things 'Green' and avoid burning through GPU cycles, I engineered a native C++ perceptual hashing (pHash) layer that acts as a bouncer[cite: 19]. When you upload a comic cover, C++ hashes it instantly at the byte level and checks the PostgreSQL cache[cite: 19]. If it's a known image, it skips the heavy Python inference entirely[cite: 19]. Honestly, it's slightly overkill for a simple comic app, but it's a great sandbox for learning enterprise-grade efficiency[cite: 19].
